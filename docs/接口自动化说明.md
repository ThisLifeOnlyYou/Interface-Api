## 核心功能

### 1. 多用户Cookie管理
- 支持在接口调用时直接指定使用哪个用户的Cookie
- Cookie格式：`token: LOGIN_USER:xxx`（如 `token: LOGIN_USER:13333333333`）
- 支持多环境配置，可通过配置文件区分线上/测试环境

### 2. 测试报告增强
- 测试报告表格显示：状态、测试用例名称、断言内容、接口数量、响应内容、时长
- 断言内容显示为：✅ 断言消息: 实际值
- 响应内容显示完整的接口响应体
- 支持中文测试用例名称显示
- 详细的断言详情和接口响应信息

### 3. 断言工具
- 提供多种断言方法：`assert_success`、`assert_fail`、`assert_business_data`、`assert_not_null`、`assert_not_empty`
- 断言信息自动记录到测试报告

## 使用方法

### 1. 安装依赖

```bash
pip install -r requirements
```

### 2. 运行测试

#### 运行所有测试

```bash
venv/bin/pytest testcases/ -v --html=reports/test_report.html
```

#### 运行指定测试文件

```bash
venv/bin/pytest testcases/test_login.py -v --html=reports/test_report.html
```

#### 运行指定测试用例

```bash
venv/bin/pytest testcases/test_login.py::TestLogin::test_login_success -v --html=reports/test_report.html
```

### 3. 在接口调用中指定用户

```python
# 使用默认用户
response = client.post(LoginPaths.LOGIN, data={"username": "13333333333", "password": "123456"})

# 指定用户（使用该用户的Cookie）
response = client.post(GoodsPaths.QUERY_PAGE, data={"page": 1, "size": 10}, user="13333333333")
```

## 测试报告

### 报告生成

测试运行完成后，会在 `reports` 目录生成 `test_report.html` 文件。

### 报告内容

- **状态**：测试用例执行状态（通过/失败）
- **测试用例名称**：显示测试用例的完整名称，支持中文
- **断言内容**：显示具体的断言信息，包括状态图标和实际值
- **接口**：显示该测试用例调用的接口数量
- **响应内容**：显示接口的响应内容，包括方法、路径、状态码和响应体
- **时长**：测试用例执行时长
- **详情**：点击可查看详细的断言信息和接口响应信息

## 配置说明

### 1. API路径配置

在 `config/api_paths.py` 中配置接口路径：

```python
class LoginPaths:
    """登录相关接口路径"""
    LOGIN = "/login"
    CHECKED_TOKEN = "/checkedToken"

class MemberPaths:
    """会员相关接口路径"""
    QUERY_PAGE = "/member_management/member/queryPageByQo"
    SAVE = "/member_management/member/save"
    UPDATE = "/member_management/member/update"
    DELETE = "/member_management/member/delete"

class GoodsPaths:
    """商品相关接口路径"""
    QUERY_PAGE = "/goods_management/goods/queryPageByQo"
    SAVE = "/goods_management/goods/save"
    UPDATE = "/goods_management/goods/update"
    DELETE = "/goods_management/goods/delete"
```

### 2. 环境配置

在 `config/settings.py` 中配置环境信息：

```python
class Config:
    """基础配置"""
    BASE_URL = "http://127.0.0.1:9291"
    TIMEOUT = 30

class TestConfig(Config):
    """测试环境配置"""
    BASE_URL = "http://127.0.0.1:9291"

class ProdConfig(Config):
    """生产环境配置"""
    BASE_URL = "http://api.example.com"

# 根据环境变量选择配置
config = TestConfig()
```

## 示例代码

### 1. 登录测试

```python
class TestLogin:
    """登录模块测试"""

    def test_login_success(self):
        """测试登录成功"""
        response = client.post(
            LoginPaths.LOGIN,
            data={
                "username": "13333333333",
                "password": "123456"
            }
        )
        # 断言响应成功
        data = assert_utils.assert_success(response)

        # 断言返回信息不为空
        assert_utils.assert_not_empty(data["data"], "employee", "employee")
        # 断言token不为null
        assert_utils.assert_not_null(data["data"], "token", "登录token")
        # 断言返回信息精准断言
        assert_utils.assert_business_data(data["data"]["employee"], "nickName", "张三11", "用户昵称")
```

## 注意事项

1. **Cookie管理**：系统会自动管理用户Cookie，无需手动处理
2. **断言信息**：所有断言都会自动记录到测试报告中
3. **响应内容**：测试报告中会显示完整的接口响应内容
4. **多环境支持**：可通过修改 `config/settings.py` 切换不同环境
5. **中文显示**：测试用例名称中的中文会正确显示在测试报告中

## 常见问题

### 1. 如何添加新的测试用例？

在 `testcases` 目录下创建新的测试文件，继承 `TestCase` 类，使用 `@pytest.mark` 装饰器标记测试用例。

### 2. 如何添加新的接口路径？

在 `config/api_paths.py` 中添加新的接口路径配置。

### 3. 如何修改环境配置？

在 `config/settings.py` 中修改相应的配置类，或通过环境变量切换配置。

### 4. 如何查看详细的测试报告？

测试运行完成后，打开 `reports/test_report.html` 文件查看详细的测试报告。

### 5. 如何指定使用某个用户的Cookie？

在调用接口时，通过 `user` 参数指定用户标识，如 `client.post(url, data={}, user="13333333333")`。
