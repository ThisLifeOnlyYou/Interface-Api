# Token 管理系统执行指南

## 快速开始

### 步骤 1: 配置 Token

编辑 `config/token_manager.py` 文件，填写您的账号信息：

```python
TOKEN_CONFIG = {
    "test": {
        "description": "测试环境",
        "accounts": [
            {
                "name": "测试账号1",
                "username": "13333333333",  # 您的测试账号
                "password": "123456",       # 您的密码
                "token": "",                # 可以留空，登录后自动填充
                "enabled": True
            }
        ]
    }
}
```

### 步骤 2: 执行测试

#### 方法一：执行单个测试文件

```bash
# 执行登录测试（会自动获取并保存 token）
venv/bin/pytest testcases/test_login.py -v

# 执行会员管理测试（会使用登录测试中保存的 token）
venv/bin/pytest testcases/test_member.py -v
```

#### 方法二：执行所有测试

```bash
# 执行所有测试（按顺序执行：先登录，再执行其他测试）
venv/bin/pytest testcases/ -v
```

#### 方法三：生成 HTML 报告

```bash
# 执行测试并生成 HTML 报告
venv/bin/pytest testcases/ -v --html=reports/report.html --self-contained-html
```

### 步骤 3: 查看测试报告

```bash
# 打开测试报告
open reports/report.html
```

## 高级用法

### 1. 指定环境执行

```bash
# 在测试代码中设置环境
# set_current_env("test")  # 测试环境
# set_current_env("prod")  # 线上环境
```

### 2. 使用不同账号执行

在测试用例中指定账号：

```python
def test_query_member(self, api_client):
    # 使用指定账号
    response = api_client.post(
        MemberPaths.QUERY_PAGE,
        account_name="测试账号1",  # 指定账号名称
        json={"currentPage": 1, "pageSize": 10}
    )
```

### 3. 手动设置 Token

如果不想通过登录获取 token，可以直接在配置文件中填写：

```python
{
    "name": "测试账号1",
    "username": "13333333333",
    "password": "123456",
    "token": "LOGIN_USER:13333333333",  # 手动填写 token
    "enabled": True
}
```

或者在代码中手动设置：

```python
from common.requests_client import client

# 手动设置 token
client.set_token("LOGIN_USER:13333333333")

# 发送请求
response = client.post(MemberPaths.QUERY_PAGE, json={...})
```

## 执行流程

### 完整执行流程

1. **配置账号信息**
   ```bash
   # 编辑配置文件
   vim config/token_manager.py
   ```

2. **执行登录测试**
   ```bash
   # 登录测试会自动获取 token 并保存
   venv/bin/pytest testcases/test_login.py::TestLogin::test_login_success -v
   ```

3. **执行其他测试**
   ```bash
   # 其他测试会使用登录测试中保存的 token
   venv/bin/pytest testcases/test_member.py -v
   ```

4. **查看测试报告**
   ```bash
   open reports/report.html
   ```

### 测试执行顺序

```
test_login.py (登录测试)
    ↓
获取 token
    ↓
保存到 token_manager.py
    ↓
test_member.py (会员管理测试)
    ↓
使用保存的 token
    ↓
执行测试
```

## 常见执行命令

### 基本命令

```bash
# 执行所有测试
venv/bin/pytest testcases/ -v

# 执行指定测试文件
venv/bin/pytest testcases/test_login.py -v

# 执行指定测试类

# 执行指定测试方法
venv/bin/pytest testcases/test_login.py::TestLogin::test_login_success -v
```

### 过滤执行

```bash
venv/bin/pytest testcases/ -v -m "smoke or p0"
```

## 故障排查

### 问题 1: Token 为空

**原因**：未执行登录测试或 token 未保存

**解决方案**：
```bash
# 先执行登录测试
venv/bin/pytest testcases/test_login.py::TestLogin::test_login_success -v

# 或手动在配置文件中填写 token
```

### 问题 2: 账号未启用

**原因**：账号的 enabled 字段为 False

**解决方案**：
```python
# 在配置文件中设置 enabled 为 True
{
    "name": "测试账号1",
    "enabled": True  # 确保为 True
}
```

### 问题 3: 环境不正确

**原因**：当前环境与预期不符

**解决方案**：
```python
# 在测试代码中设置正确的环境
from config.token_manager import set_current_env
set_current_env("test")  # 或 "prod"
```

## 最佳实践

1. **执行顺序**：先执行登录测试，再执行其他测试
2. **环境隔离**：测试环境和线上环境使用不同的账号
3. **Token 管理**：定期更新 token，避免过期
4. **日志检查**：定期检查日志，确保 token 正确使用
5. **报告查看**：每次执行后查看报告，确保测试通过
